# Languages

Reaching people in their native language is a project goal, not an afterthought. The
architecture is ready from day one; translation WAVES open at Phase 4 when the English
text stabilizes (translating churning text burns volunteers). Do not submit translations
before then. This file fixes the conventions so every string written today survives
translation tomorrow.

## How it works

- `data/strings/en.json` is the single source of truth for every displayed string.
  Flat dot-keys (`event.gridStrain.title`), plain string values.
- A language is one file: `data/strings/de.json` with the same keys. Missing keys fall
  back to English at runtime and are listed by `pnpm validate`, so partial translations
  ship gracefully and honestly.
- Game data never contains display text, only keys. Ids, tags, and flags are internal
  identifiers and are never translated.
- The paper kit renders from the same strings map, so a locale file also yields a
  print-and-play kit in that language. Classrooms get this for free.

## Conventions (binding for all new strings)

1. **Whole sentences only.** Never build prose by concatenating string parts in code;
   word order differs across languages. Compact game notation on cards and tracks
   ("Trust +1") is exempt: it is notation, not prose.
2. **Interpolation** uses single-brace named placeholders: `"Turn {turn} of {maxTurns}"`.
   Translators may reorder placeholders; code never assumes their position.
3. **Plurals** use key variants resolved via `Intl.PluralRules`:
   `run.turnsLeft.one` / `run.turnsLeft.other` (locales add the categories they need,
   e.g. `.few` for Polish). No `count === 1` ternaries in code.
4. **Numbers and dates** format through `Intl.NumberFormat` / locale-aware templates,
   never hand-assembled.
5. **No mid-string markup.** If part of a sentence needs styling, split at natural
   sentence boundaries or use placeholder-wrapped segments, never `<b>` inside values.
6. **Voice carries over.** Translations keep the register: short sentences, plain words,
   serious about stakes, dry about absurdity. A translation that reads like a press
   release is a bug even when it is accurate.
7. **RTL and CJK readiness** is a UI obligation: layout uses CSS logical properties,
   and the shell sets `lang` and `dir` from the active locale.

## Translation waves (Phase 4)

Per locale: a machine-assisted first pass reviewed line-by-line by a native speaker
before shipping (both credited). English string changes after a wave are tracked by
key, and stale translated keys fall back to English rather than showing outdated text.
The exact tooling (in-repo PRs vs a hosted platform like Weblate) gets decided when the
first wave starts, with community input.
